This file lists the API endpoints (method + path) found in server route files and their expected auth requirements.

Auth and base mount:
- app mounts superadmin routes at /api/superadmin and also /superadmin
- bused base for public/api routes at /api/*

Endpoints (summary):

// Auth
POST /api/auth/register
POST /api/auth/login
GET /api/auth/debug (protected)

// Booking routes (/api/bookings)
POST /api/bookings/    (protect, authorize user) - create booking
POST /api/bookings/batch (protect, user) - batch create
GET  /api/bookings/my-bookings (protect, user)
GET  /api/bookings/turf/:turfId (public) - get bookings for turf (date filter)
GET  /api/bookings/:id (protect) - get booking by id
GET  /api/bookings/:id/invoice (protect) - invoice
GET  /api/bookings/turfadmin (protect, admin|turfadmin) - turfadmin bookings for their turfs
PUT  /api/bookings/:id/status (protect, admin|superadmin) - update status
POST /api/bookings/:bookingId/release (protect, admin|superadmin)
POST /api/bookings/cleanup-pending (protect, superadmin)
GET  /api/bookings/audit-logs (protect, admin|superadmin)
GET  /api/bookings/all (protect, superadmin) - get all bookings

// Payments (/api/payments)
POST /api/payments/create-order (protect, user)
GET  /api/payments/user (protect, user)
POST /api/payments/validate-keys (protect, superadmin|admin|turfadmin)
POST /api/payments/dev/complete (dev only)
POST /api/payments/verify (protect, user)

// Turf routes (/api/turfs)
GET  /api/turfs/ (public)
GET  /api/turfs/:id (public)
POST /api/turfs/ (protect, admin|turfadmin)
GET  /api/turfs/my-turfs (protect, admin|turfadmin)
PUT  /api/turfs/:id (protect, admin|turfadmin|superadmin) - controller enforces ownership
DELETE /api/turfs/:id (protect, admin|turfadmin|superadmin)
PATCH /api/turfs/:id/approve (protect, superadmin)
PATCH /api/turfs/:id/block (protect, superadmin)

// Turfadmin (/api/turfadmin)

// User (/api/user)

Notes:
- Many endpoints are protected and require a valid JWT; see middleware in server/middleware/authMiddleware.js
- To test endpoints: include Authorization: Bearer <token> header or cookies
- The dev server may also expose helper endpoints (e.g., payments dev/complete) in non-production
